## Estándares de Calidad de Documentación

### Criterios para Documentación de Alta Calidad

```
✅ Tiene sección de Introducción clara
✅ Tiene tabla de contenidos (para docs > 500 líneas)
✅ Diagrama Mermaid donde aplique
✅ Código de ejemplo o comandos reales
✅ Tabla de variables/parámetros
✅ Sección de troubleshooting o errores comunes
✅ Notas de seguridad donde aplique
✅ Cross-references a documentos relacionados
✅ Estado real vs simulado claramente marcado
✅ Metadata de versión y fecha
```

### Criterios de Calidad por Tipo

| Tipo de Documento | Criterios Mínimos |
|---|---|
| Guía de Instalación | Prerrequisitos, comandos paso a paso, verificación |
| Referencia de API | Método, URL, auth, params, responses, ejemplos |
| Arquitectura | Diagrama, descripción de componentes, dependencias |
| Runbook Operacional | Procedimiento paso a paso, verificaciones, rollback |
| Documentación de Negocio | Propuesta de valor, audiencia, casos de uso |
